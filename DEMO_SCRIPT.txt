[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's trace the architecture. When a farmer sends a query, the workflow starts by loading their stored profile from our JSON Session Memory node. The query is then analyzed by the Security Agent, which scrubs location coordinates, detects prompt injections, and logs a structured JSON audit trail.

If passed, the query goes to the orchestrator agent. The orchestrator delegates reasoning dynamically to three sub-agents: the Crop Doctor, the Weather Agent, and the Market Analyst. These sub-agents query a local Model Context Protocol server, which connects directly to the live Open-Meteo weather API. Every sub-agent outputs its findings along with a confidence score.

The orchestrator compiles a draft plan. If hazardous chemicals are recommended, a Human Review node pauses the workflow and requests approval from the farmer. Finally, the Recommendation Agent structures all advice into a strict JSON schema, which is translated by our Bangla Translation node into fluent Bangla.
